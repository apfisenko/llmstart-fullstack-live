## Решения

- Таблица этапов названа **`progress_checkpoints`**, не `lessons`, чтобы совпадать с OpenAPI и доменным термином; продуктовый «урок» = строка в этой таблице.
- Используются **`cohorts` / `cohort_memberships`**, без введения `flows`/`enrollments` в БД без ADR.
- Переписка с ассистентом в БД — **`dialogue_turns`**: одна строка на пару Q&A; два UUID (`id`, `assistant_message_id`) для совместимости с формой ответа `PostDialogueMessage` в OpenAPI.

## Отклонения / долг

- Поле `is_homework` пока не добавлено в схему `ProgressCheckpointItem` в OpenAPI — сознательно отложено до отдельного изменения контракта.
- Связь «вопрос ↔ урок» для статистики по урокам не вводилась (как в сценариях — задел на следующие итерации).
