# Технический долг: frontend (`frontend/web`)

**Обновлено:** 2026-04-17  
**Основание проверки:** ориентиры из skills `shadcn-ui`, `vercel-react-best-practice`; перечень релевантных project skills через `find-skills` (в т.ч. `nextjs-app-router-patterns`, `sharp-edges` — на будущее при доработке маршрутов и окружений).

---

## P1 — высокий приоритет

1. **Широкая граница `"use client"` на страницах**  
   Маршруты `app/page.tsx`, `app/login/page.tsx`, `app/(app)/teacher/page.tsx`, `app/(app)/leaderboard/page.tsx`, `app/(app)/chat/page.tsx` объявлены целиком как клиентские. По практикам Vercel/React (App Router) предпочтительнее держать страницы серверными и выносить интерактив в дочерние клиентские «листья», чтобы уменьшить JS на клиенте и упростить последующий SSR данных.

2. **Клиентский каркас `(app)/layout` через `AppShell`**  
   Весь сегмент `(app)` оборачивается в клиентский `AppShell` (навигация, сессия, виджет чата). Это осознанно для `localStorage` и `usePathname`, но фиксирует архитектурный долг: любые тяжёлые серверные сценарии для «оболочки» без рефакторинга (разделение: серверный layout + тонкие клиентские острова) усложняются.

3. **Сессия только в `localStorage` и редиректы в `useEffect`**  
   Нет `middleware` и httpOnly-cookie-модели. Защита маршрутов и редиректы (`/`, `/login`, `AppShell`) выполняются на клиенте после гидрации — возможны вспышки «Загрузка…», лишний клиентский код и ограничения для SEO/безопасности по сравнению с серверной проверкой сессии. Для dev-сценария приемлемо; для продукта — отдельное решение (см. также skill `sharp-edges`).

## P2 — средний приоритет

4. **Нет сегментных `loading.tsx` / `error.tsx`**  
   В `app/` отсутствуют границы Suspense и обработки ошибок на уровне сегментов — рекомендация App Router для предсказуемого UX при медленных запросах и сбоях.

5. **Отсутствует `components.json` (shadcn CLI)**  
   Тема и `@import "shadcn/tailwind.css"` используются, UI-слой согласован с токенами и `cn()`, но нет файла конфигурации shadcn CLI — новым разработчикам сложнее повторять официальный поток `shadcn add` и алиасы; возможен дрейф стиля при ручном копировании компонентов.

6. **Стек примитивов ≠ классический shadcn (Radix)**  
   Интерактивные части (`Button`, `Input`, `Sheet`) завязаны на `@base-ui/react`, а не на Radix из типичного реестра shadcn. Это не баг, но расхождение с текстом skill `shadcn-ui` — при добавлении компонентов из реестра нужно сверять API (слоты, `asChild`, доступность) и не смешивать паттерны вслепую.

7. **`AppShell`: отложенная готовность через `setTimeout(..., 0)`**  
   После чтения сессии состояние `ready` выставляется асинхронно на следующий тик — лишнее мигание экрана «Загрузка…» без явной необходимости (можно упростить поток инициализации).

## P3 — низкий приоритет / полировка

8. **Жёстко заданные цвета статусов в матрице преподавателя**  
   В `teacher/page.tsx` для ячеек используются утилиты вроде `emerald-500`, `amber-500`. Skill `shadcn-ui` рекомендует опираться на семантические токены темы (`chart-*`, `muted`, кастомные статус-токены), чтобы не расходиться с тёмной темой и единым языком дизайна.

9. **BFF `POST` `auth/dev-session`: upstream `fetch` без явного `cache: "no-store"`**  
   Остальные прокси-роуты чаще задают `cache: "no-store"`. Для `POST` риск кэширования ниже, но единообразие снизит сомнения при апгрейдах Next.

10. **Нет автотестов / CI-гейта только под frontend**  
   В `package.json` есть `lint`, нет скрипта тестов (Playwright/Vitest и т.д.) — регрессии UI и контрактов с BFF остаются на ручной проверке.

11. **`next/image` не используется в исходниках приложения**  
   На текущем UI картинок может не быть; при появлении медиа стоит перейти на `next/image` с корректными `sizes` (рекомендация skill `vercel-react-best-practice`).

## Кратко: что уже в хорошем состоянии

- Корневой `app/layout.tsx` — RSC, метаданные, `next/font`, `lang="ru"`.  
- Прокси к backend в Route Handlers с явным `cache: "no-store"` для GET-данных (дашборд, лидерборд, turns и т.д.).  
- Клиентские `fetch` к BFF для чата/дашборда с `cache: "no-store"` где проверено.  
- Утилита `cn()`, семантические CSS-переменные темы, CVA для кнопок — соответствуют ожиданиям shadcn-стека.  
- Страница `app/(app)/student/page.tsx` остаётся серверным компонентом (контраст с соседними маршрутами).
