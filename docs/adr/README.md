# Архитектурные решения (ADR)

Здесь фиксируются **значимые** и **устойчивые** решения по архитектуре и платформе: что выбрано, почему, какие были альтернативы и какие последствия. Детали реализации остаются в коде и в специализированных документах (`data-model.md`, `vision.md` и т.д.).

## Принципы

- **Один ADR — одно решение** (или тесно связанный пакет, который нельзя откатить по частям).
- **Статус** в шапке: черновик / принято / устарело / заменено (со ссылкой на новый ADR).
- **Контекст** — зачем решение понадобилось; **решение** — что именно принято; **альтернативы** — что отвергнуто и почему; **последствия** — плюсы, минусы, риски.
- ADR **не переписывают задним числом**: при смене решения добавляется новый ADR или помечается устаревание с отсылкой к преемнику.
- Нумерация **последовательная**: `adr-NNN-short-title.md`.
