Создай `docs/tasks/tasklist-frontend.md` на основе шаблона `@docs/templates/tasklist.md` для реализации frontend-части проекта.

Технологический стек:
- Framework: Next.js (App Router) + React
- Язык: TypeScript
- UI Library: shadcn/ui
- Styling: Tailwind CSS
- Пакетный менеджер: pnpm

Последовательность задач:
Задача 1: Требования к UI и API-контракты для frontend
- Сформировать краткие функциональные требования к четырём зонам интерфейса:
  - Экран 1 — панель преподавателя: 4 KPI с дельтой; график активности по дням (14 дней); лента вопросов (кто, когда, вопрос + ответ); лента сдач (кликабельно — отчёт и ссылки); матрица прогресса (студенты × уроки, дата при наведении)
  - Экран 2 — лидерборд: переключатель таблица / scatter plot; таблица — место, прогресс-бар, иконки по урокам, топ-3 с медалями; 
  - Глобальный чат: плавающая кнопка на всех экранах
- Зафиксировать общий стиль UI: тёмная тема, dev стиль (ближе к https://www.tbench.ai/)
- Зафиксировать вход: без сложной авторизации — ввод Telegram username
- Спроектировать API-контракты для всех экранов

Задача 2: Реализация API для frontend
- Проверить достаточность структуры данных для реализации экранов и новых API и сформулировать требования к данным для реализации
- Реализовать новые endpoint'ы в backend, необходимые для отрисовки всех экранов, включая доработку структуры данных если необходимо
- Сформировать миграцию с mock данными для красивого наполнения frontend данными
- Добавить миграцию добавления преподавателя в базу данных (@akozhin Id: 162684825)
- Проверка через skill `api-design-principles`
- Обновить `docs/tech/api-contracts.md`

Задача 3: Каркас frontend-проекта
- Использовать skills `shadcn` и `vercel-react-best-practices` и `nextjs-app-router-patterns`
- Инициализировать проект: Next.js (App Router) + TypeScript + shadcn/ui + Tailwind CSS + pnpm
- Настроить тему проекта
- Реализовать вход (отображение информации о пользователе и кнопку выхода)
- Настроить базовый layout (навигация по экранам, глобальный чат-виджет)
- Настроить автоматизацию запуска проекта

Задача 4: Панель преподавателя
- Реализовать страницу дашборда преподавателя
- 4 KPI-карточки
- Линейный график активности
- Таблица вопросов
- Живая лента последних сдач
- Матрица прогресса

Задача 5: Лидерборд
- Реализовать страницу лидерборда
- Переключатель «Таблица / Карта» 

Задача 6: Чат с ассистентом
- Реализовать страницу чата с ассистентом
- История переписки

Задача 7: Чат в основной области (меню «Чат»)
- Реализовать страницу чата в основной области
- История переписки

Задача 8: Ревью качества кода frontend
- Проверить код на соответствие best practices (skills `vercel-react-best-practices` и `nextjs-app-router-patterns`)
- Зафиксировать и исправить критические ошибки и недоработки

Задача 9: Голосовой режим чата
- Реализовать голосовой режим чата
  - В веб-версии
  - В телеграм-версии

Задача 10: Добавить возможность отвечать на вопросы по данным из базы данных
- Проработать и зафиксировать варианты реализации (Text-to-SQL)
- Реализовать в соответствии с принятой архитектурой
- Подготовить сценарии для тестирования

Там, где уместно, добавь в состав работ актуализацию проектной документации с перечислением конкретных файлов.

Внутри итераций используй `Definition of Done` в двух вариантах:
- короткий список для самопроверки агента: что должно быть сделано и чему результат должен соответствовать
- короткий список для проверки пользователем: что открыть, что запустить, что проверить руками

Если в ходе работы появляются новые команды локального запуска или проверки, добавляй или актуализируй соответствующие команды там, где это уместно. Запуск всех команд через powershell, для работы с docker используется wsl

Задачи формулируй кратко и предметно.
