# Sharp edges (подводные камни)

## Принцип

- **Проверять инварианты**: что должно быть истиной в рантайме (один запрос — одна сессия БД, один контекст транзакции, одна версия схемы) и где оно ломается.
- **Не смешивать модели**: sync-драйвер/ORM-вызовы внутри async-петли без `to_thread` — блокировка event loop; наоборот — лишние потоки и гонки.

## Backend (FastAPI + SQLAlchemy async)

- Сессия БД — **на границе запроса** (`Depends`); не хранить сессию в глобалах; не переиспользовать объект сессии между запросами.
- **Lazy loading** связей после закрытия сессии — классический `DetachedInstanceError` / неожиданные N+1; явно грузить нужное (`selectinload` / eager) в том же запросе, где живёт сессия.
- **Порядок зависимостей** и side effects в `Depends` — порядок выполнения важен; циклические зависимости ломают старт приложения.
- **Параллельные `commit`/`flush`** на одной сессии — не допускать; один writer-поток на сессию.
- **Pydantic Settings**: обязательные поля без дефолтов — «падает при старте» vs «падает в рантайме при первом обращении» — разные классы багов.

## LLM и логи (этот репозиторий)

- **Не логировать** тексты пользователя и ответы модели — при отладке использовать **длины**, **хэши**, **идентификаторы** диалога/запроса, коды ошибок провайдера.
- Таймауты, ретраи и **идемпотентность** записи в БД при повторе запроса к LLM.

## Next.js / React (кратко)

- Проп из **Server Component** в **Client** — только сериализуемые данные; функции, классы, символы — нет.
- **`useSearchParams`** и подобное — клиент; не тянуть в корень без `Suspense`, если это важно для границ (см. доку версии).
- Кэш **`fetch`** / сегментов — «вижу старые данные» часто из политики кэша, а не из «сломанного» UI.

## Окружение и данные

- **Часовые пояса**: UTC в хранении, явный формат при сериализации; не полагаться на локаль сервера.
- **Плавающие** деньги и сравнения float — для денег типы/decimal в БД и в домене.
- **Кодировки и пути** на Windows vs CI (Linux) — регистр файлов, `\` vs `/`, `.env` не в репо.

## Когда читать этот skill

- Баг **воспроизводится не всегда** или только на **Vercel/CI**.
- «Локально работает» при отличии **env**, **версии Node/Python**, **миграций**.
- После **апгрейда** Next/FastAPI/SQLAlchemy — сравнить changelog с паттерном в коде.
