# shadcn/ui

## Суть

- Компоненты **копируются в репозиторий** (`components/ui/…`), а не «чёрный ящик» из npm — правки и кастом допустимы по умолчанию.
- База: **Radix Primitives** (доступность, поведение) + **Tailwind CSS** (внешний вид) + часто **class-variance-authority** для вариантов.

## CLI (актуальная линия)

- Инициализация: `npx shadcn@latest init` (или `pnpm dlx` / `yarn dlx` по стеку проекта).
- Добавление: `npx shadcn@latest add <component>` — не переписывать вручную то, что уже даёт реестр, если задача — стандартный паттерн.
- Следовать **`components.json`** проекта: алиасы путей (`@/components`), стиль (default/new-york), базовый цвет, CSS variables.

## Код

- Утилита **`cn()`** (обычно `clsx` + `tailwind-merge`): объединять классы без конфликтов; для вариантов кнопок/бейджей — **CVA** (`cva`, `VariantProps`), как в шаблонах shadcn.
- **Композиция**: предпочитать слоты и пропсы `asChild` там, где компонент оборачивает `Slot` из `@radix-ui/react-slot` — один корневой элемент без лишних DOM-обёрток.
- **Тема**: опираться на CSS variables из темы shadcn (`background`, `foreground`, `primary`, `muted`, `border`, `radius` и т.д.), не хардкодить произвольные hex там, где уже есть семантические токены.
- **Доступность**: не отключать фокус-стили; сохранять `aria-*` и поведение Radix при обёртках; кликабельные области — адекватный размер и контраст.

## Интеграция в проект

- Согласовать с существующим фронтом: React/Next/Vite, версия Tailwind, путь к `tailwind.config`, глобальные стили (`globals.css`).
- После `add` проверить импорты алиасов (`@/lib/utils`, `@/components/ui/...`) и зависимости, которые подтянул CLI.
- Не тащить лишние компоненты «на будущее» — только нужные для задачи (YAGNI).
