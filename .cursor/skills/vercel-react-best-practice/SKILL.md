# Vercel + React (практики)

## Базовая модель (App Router)

- По умолчанию модули — **Server Components**: нет лишнего JS на клиенте, прямой доступ к БД/секретам через серверный код (без утечки в бандл).
- **`"use client"`** — только там, где нужны хуки React, обработчики событий, браузерные API, состояние UI. Держать границу **узкой**: не оборачивать всё приложение без причины.
- **Композиция**: серверный layout/page импортирует клиентские «листья»; пропсы из RSC в клиент должны быть **сериализуемы** (без функций/классов DOM из сервера в клиент как проп).

## Данные и кэш

- В RSC предпочитать **`async` компоненты** и `fetch` на сервере с явной политикой кэша (`cache`, `next: { revalidate }`, `no-store`) — осознанно, под требования свежести данных.
- Не смешивать «истину» с клиентским стейтом без нужды: **сервер — источник** для начального рендера; клиент — для интерактива.
- **Server Actions** — для мутаций из форм/кнопок с валидацией и revalidation (`revalidatePath` / `revalidateTag`) по смыслу задачи, не как замена REST без причины.

## Маршруты и UX

- **`loading.tsx`** / **`error.tsx`** / **`not-found`** — по месту сегмента для предсказуемого UX и границ Suspense.
- **Параллельные и перехватывающие маршруты** — только когда реально упрощают модальные сценарии; иначе KISS.

## Производительность

- **`next/image`** для картинок с корректными `sizes` при responsive; **шрифты** через `next/font` где уместно.
- **`dynamic(() => import(...), { ssr: false })`** — только при жёсткой необходимости (нет SSR для модуля); по умолчанию избегать лишнего client-only.
- Избегать тяжёлых зависимостей в клиентских корнях; **анализ бандла** при росте страницы.

## Vercel (деплой)

- Секреты и конфиг — **env в панели Vercel** / `.env.local`; не хардкодить ключи; для preview/production различать переменные осознанно.
- **Edge vs Node** — выбирать runtime по нужде (Node-API, длительность, streaming); не включать edge «по умолчанию» без проверки совместимости зависимостей.

## Актуальность

- Детали API Next.js и платформы Vercel меняются между мажорными версиями — при сомнениях сверяться с официальными гайдами: [https://nextjs.org/docs](https://nextjs.org/docs), [https://vercel.com/docs](https://vercel.com/docs).
